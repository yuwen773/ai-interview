# 模拟面试页面 - 答题卡抽屉交互优化

## 背景问题

当前面试页面中，`messages` 数组随着答题数量线性增长（每题增加两条消息：面试官问题和用户回答），导致：
- 页面被历史记录撑长
- 用户无法专注当前题目
- 小程序端体验差，信息密度过高

## 设计目标

将**历史记录**和**当前答题**解耦，提供答题卡导航，在保持专注的同时保留完整的历史导航能力。

## 数据结构

### 题目状态枚举

```ts
type QuestionStatus = 'answered' | 'saved' | 'unanswered';
// answered = 🟢 已答（已提交）
// saved   = 🟡 暂存（可修改）
// unanswered = 🔴 未答
```

### 答题卡条目

```ts
interface AnswerCardItem {
  questionIndex: number;    // 题号（1-based）
  status: QuestionStatus;
  question?: string;        // 点击展开时显示
  savedAnswer?: string;     // 暂存的答案（🟡时可修改）
}
```

### 状态规则

| 操作 | 状态变化 |
|------|---------|
| 进入面试（首题） | 首题 → 🔴 未答 |
| 点击"暂存答案" | 该题 → 🟡 暂存 |
| 点击"提交答案" | 该题 → 🟢 已答，下一题 → 🔴 未答 |
| 恢复会话（已有答案） | 保留原有状态 |

## 技术实现要点

### 手势处理

使用 Taro 的 `onTouchStart` / `onTouchMove` / `onTouchEnd` 事件监听手势：
- 水平滑动距离 > 50px 且方向为从右向左 → 打开抽屉
- 水平滑动距离 > 50px 且方向为从左向右 → 关闭抽屉

### 状态管理

答题卡数据从 `session.questions` 派生：

```ts
const answerCards: AnswerCardItem[] = session.questions.map((q, index) => ({
  questionIndex: index + 1,
  status: index < session.currentQuestionIndex
    ? 'answered'
    : index === session.currentQuestionIndex && q.userAnswer
      ? q.userAnswer.trim() ? 'answered' : 'saved'
      : 'unanswered',
  question: q.question,
  savedAnswer: q.userAnswer,
}));
```

### 暂存修改流程

用户点击🟡条目展开后修改答案 → 点击"保存" → 调用 `saveAnswer` API → 更新本地状态

## 设计优势

1. **专注当前** — 页面不再被历史记录撑长，用户始终聚焦当前题目
2. **快速导航** — 答题卡提供全局视图，快速跳转到任意题目
3. **清晰状态** — 三色状态一眼辨别进度
4. **移动端友好** — 右侧滑动手势符合直觉，60%宽度保留操作空间
