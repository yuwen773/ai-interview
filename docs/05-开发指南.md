# 开发指南

## 1. 环境准备

### 1.1 基础环境要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 21+ | 必须启用虚拟线程 |
| Maven | 3.9+ | 构建工具 |
| Node.js | 18+ | 推荐 20 LTS |
| PostgreSQL | 15+ | 需支持 pgvector |
| Redis | 7+ | 缓存和消息队列 |
| Git | 最新版 | 版本控制 |

### 1.2 本地开发环境搭建

#### 步骤 1: 克隆项目

```bash
git clone <repository-url>
cd ai-interview
```

#### 步骤 2: 配置数据库

```bash
# 启动 PostgreSQL (使用 Docker)
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=ai_interview \
  -p 5432:5432 \
  pgvector/pgvector:pg16

# 启用 pgvector 扩展
docker exec -it postgres psql -U postgres -d ai_interview -c "CREATE EXTENSION vector;"
```

#### 步骤 3: 启动 Redis

```bash
docker run -d \
  --name redis \
  -p 6379:6379 \
  redis:7-alpine
```

#### 步骤 4: 配置环境变量

复制并编辑配置文件：

```bash
# 后端配置
cp app/src/main/resources/application-local.example.yml app/src/main/resources/application-local.yml

# 前端配置
cp frontend/.env.example frontend/.env.local
```

编辑 `application-local.yml`：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/ai_interview
    username: postgres
    password: postgres

  data:
    redis:
      host: localhost
      port: 6379

ai:
  bailian:
    api-key: ${AI_BAILIAN_API_KEY}

app:
  storage:
    endpoint: http://localhost:9000
    access-key: minioadmin
    secret-key: minioadmin
```

### 3.3 Git 提交规范

使用约定式提交格式：

```

```

**类型 (type)**:
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关

## 6. 常见问题

### 6.1 后端问题

**Q: 启动失败，提示连接数据库失败**

A: 检查 PostgreSQL 是否启动，配置是否正确

```bash
# 测试连接
psql -h localhost -U postgres -d ai_interview
```

**Q: AI 调用超时**

A: 检查网络连接和 API Key 配置

```yaml
# application-local.yml
ai:
  bailian:
    api-key: ${AI_BAILIAN_API_KEY}  # 确保环境变量已设置
```

**Q: Redis Stream 消费者不工作**

A: 检查消费者组是否创建

```bash
redis-cli
> XGROUP CREATE resume:analyze:stream analyze-group 0 MKSTREAM
```

### 6.2 前端问题

**Q: npm install 失败**

A: 尝试使用国内镜像

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

**Q: 接口请求跨域**

A: 后端已配置 CORS，确保前端代理配置正确

```typescript
// vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
      }
    }
  }
});
```

### 6.3 Docker 问题

**Q: 容器启动失败**

A: 查看详细日志

```bash
docker-compose logs app
docker-compose logs postgres
```

**Q: 端口冲突**

A: 修改 docker-compose.yml 中的端口映射

```yaml
services:
  app:
    ports:
      - "8081:8080"  # 改为 8081
```

## 7. 性能优化建议

### 7.1 数据库优化

- 添加必要的索引
- 使用查询优化 (避免 N+1)
- 启用连接池

### 7.2 缓存优化

- 热点数据缓存
- 合理设置 TTL
- 使用缓存预热

### 7.3 AI 调用优化

- 批量处理
- 结果缓存
- 降级策略

## 8. 安全建议

- 不要在代码中硬编码密钥
- 使用环境变量管理敏感配置
- 定期更新依赖版本
- 添加输入验证和输出转义
