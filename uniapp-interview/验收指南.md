# AI 模拟面试小程序验收指南

本指南旨在为 AI 模拟面试小程序提供完整的功能验收标准，覆盖从首页到面试复盘的全链路闭环。

---

## 1. 首页 (Dashboard)
首页是用户的训练入口，应根据用户的不同进度展示对应的“首要任务卡片”。

- [ ] **摘要卡片加载**：打开首页，检查“首要任务”卡片是否能根据当前数据状态正常加载。
- [ ] **状态覆盖验证**（需覆盖以下至少一种状态）：
    - **无简历**：展示“新手起步”卡片，引导上传简历。
    - **简历分析中**：展示简历解析进度。
    - **有未完成面试**：展示“优先完成”卡片，显示继续面试入口。
    - **有最近报告**：展示最近一次面试的分数及查看报告入口。
- [ ] **快捷入口跳转**：验证页面中部的快捷入口（简历库、面试记录、成长曲线）是否能正确跳转到对应页面。

## 2. 简历库 (Resume Library)
管理所有已上传的简历档案。

- [ ] **列表展示**：检查列表项是否包含：文件名、上传时间、文件大小、最近得分、面试次数。
- [ ] **功能按钮**：验证“去上传新简历”、“查看详情”、“开始面试”、“删除”按钮功能是否正常。
- [ ] **空态检查**：若无数据，验证是否展示空态文案及“上传简历”引导入口。

## 3. 上传与简历详情 (Upload & Detail)
简历的解析与深度分析展示。

- [ ] **上传流程**：点击上传，选择文件后验证是否自动进入“简历详情页”。
- [ ] **详情页展示**：
    - **分析状态**：展示 PENDING/PROCESSING/FAILED/COMPLETED 状态。
    - **分析摘要**：展示简历的核心背景摘要。
    - **维度得分**：展示各能力维度的得分条。
    - **亮点建议**：展示系统生成的亮点总结和改进建议列表。
    - **关联面试**：展示该简历下所有的面试记录列表。
- [ ] **操作验证**：验证“开始面试”、“重新分析”、“导出 PDF”、“查看成长曲线”按钮。
- [ ] **异步轮询**：若分析在进行中，确认页面每 3 秒自动刷新，且状态变化（如 PROCESSING -> COMPLETED）能实时体现。

## 4. 岗位配置 (Interview Config)
面试前的参数设定。

- [ ] **简历摘要**：检查当前简历的名称和文本摘要是否展示正确。
- [ ] **岗位选择**：检查岗位卡片列表是否可选，选中态是否清晰。
- [ ] **恢复路径**：如果该简历已有未完成面试，验证页面是否弹出“检测到未完成面试”区块，并提供“继续该面试”和“开始新的”两个路径。
- [ ] **创建会话**：点击开始，验证是否成功进入面试进行页。

## 5. 面试进行页 (Interview Session)
核心模拟面试交互体验。

- [ ] **界面元素**：
    - **顶部进度**：展示当前题号/总题数。
    - **问题卡片**：清晰展示 AI 提出的当前问题。
    - **问答流**：可滚动查看之前的历史问答记录。
    - **输入区域**：支持多行文本输入。
- [ ] **动作验证**：
    - **提交答案**：点击提交后，验证是否进入下一题。
    - **暂存答案**：点击暂存，验证是否保存当前输入且不切换题目。
    - **提前完成**：点击后弹出确认框，确认后验证是否直接进入报告生成流程。

## 6. 面试报告页 (Interview Report)
面试后的全方位评估。

- [ ] **Hero 区域**：展示总分、本场结论（如“表现稳定”）。
- [ ] **分类得分**：展示按题型聚合的维度得分条（颜色应随分数变化，如 85+ 为绿色）。
- [ ] **总体评价**：展示 AI 对本场表现的详细评价。
- [ ] **逐题反馈**：检查每道题的题目、我的回答、AI 评价（得分、优点、改进建议）。
- [ ] **参考答案**：检查每道题是否配有“参考答案/关键点”。
- [ ] **导出 PDF**：验证点击导出后是否能正常生成并打开 PDF 文件。
- [ ] **处理中状态**：若报告尚未生成完成，检查是否展示合理的加载中或空态提示。

## 7. 面试记录 (Interview History)
历史训练数据的聚合。

- [ ] **顶部统计**：检查总场次、已完成数、平均分统计是否准确。
- [ ] **列表信息**：检查每项是否展示：简历名、岗位名、状态（进行中/已完成）、分数、创建/完成时间。
- [ ] **动作验证**：
    - 进行中记录 -> 点击跳转至面试页继续。
    - 已完成记录 -> 点击跳转至报告页。
    - 支持“导出 PDF”和“删除”操作。

## 8. 成长曲线 (Growth Curve)
长期进步趋势的可视化。

- [ ] **岗位切换**：检查顶部的岗位切换 Chips 是否能正常过滤数据。
- [ ] **趋势图表**：检查柱状图是否随岗位切换而更新，日期与分数对应准确。
- [ ] **核心指标**：检查最近得分、最佳得分、有效记录数。
- [ ] **分类表现**：展示最近一次练习的各维度得分。
- [ ] **空态检查**：无历史数据时，验证是否展示“暂无统计”及“开始面试”引导。

## 9. 我的 (Profile)
产品信息与反馈。

- [ ] **功能展示**：验证不再是占位页，包含产品定位、操作入口（产品说明、帮助反馈、继续训练）。
- [ ] **版本信息**：检查底部是否正确展示小程序版本（v1.0.0）、运行框架及构建目标。
- [ ] **反馈弹窗**：点击“反馈说明”，验证是否弹出引导反馈的 Modal。

## 10. 全局通用要求 (General)
- [ ] **文案编码**：全站中文文案无乱码，语境统一（如“面试”与“练习”术语一致）。
- [ ] **交互一致性**：加载中 (Loading) 和空态 (Empty) 组件风格统一。
- [ ] **核心闭环**：确保“上传 -> 分析 -> 配置 -> 面试 -> 报告 -> 记录 -> 曲线”的完整链路完全跑通，无死循环或中断。
